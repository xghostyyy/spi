# SPI Messenger — Техническое задание (v1.0)

**Проект:** веб-мессенджер «SPI Messenger» (кратко — SPI)
**Заказчик стилистики:** ООО «Стройпутьизыскания» (https://spi-2015.ru/)
**Дата:** 02.07.2026

---

## 1. Общее описание

SPI — кроссплатформенный веб-мессенджер, работающий одинаково хорошо на ПК и мобильных устройствах. Основной сценарий использования — **мобильный (iPhone)**: приложение устанавливается на домашний экран как PWA, получает push-уведомления и внешне не отличается от нативного приложения.

### 1.1 Целевые платформы

| Платформа | Режим | Приоритет |
|---|---|---|
| iPhone (Safari, iOS 16.4+) | PWA на домашнем экране, push | **Высший** |
| Android (Chrome) | PWA, push | Высокий |
| ПК (Chrome/Edge/Firefox/Safari) | Полноэкранная веб-версия (двухпанельный макет) | Высокий |
| Планшеты | Адаптивный макет | Средний |

### 1.2 Дизайн

- Внешний вид строго по макетам Figma (приложены изображениями): десктопная версия — двухпанельная (список чатов слева, диалог справа), мобильная — одноэкранная навигация (список → чат).
- Фирменные цвета: **белый, оранжевый, тёмно-синий (ближе к чёрному)** — по стилистике сайта spi-2015.ru.
- Две темы: **светлая** (белый + оранжевый + тёмно-синий) и **тёмная** (как на макетах). Переключение в настройках + автоопределение по системной теме.
- Дизайн-токены (уточнить пипеткой по макетам/сайту):
  - `--spi-orange: #E87B2D` — акцент (кнопки, активные элементы, бейджи непрочитанных)
  - `--spi-navy: #10233F` — тёмно-синий (заголовки, свои пузыри сообщений в светлой теме)
  - `--spi-white: #FFFFFF`
  - Тёмная тема: фоны `#181B1F / #1F2328 / #2A2F35`, пузыри `#3A4046` (входящие) / `#C9CDD1`-светлые (исходящие, как на макете)
- Скругления крупные (пузыри ~16px, инпуты — pill), плотность интерфейса как на макетах.
- Учет `safe-area-inset` для iPhone (чёлка/индикатор Home).

## 2. Функциональные требования

### 2.1 Аккаунт и вход
- **Регистрация и вход через e-mail**: одноразовый код на почту (без пароля) + опциональный пароль.
- JWT access (короткоживущий) + refresh-токен (httpOnly cookie), ротация refresh-токенов.
- Сессии: список активных устройств, завершение отдельной сессии/всех сессий.
- **Username** (@nickname, уникальный) — добавление в контакты по username, номер телефона скрыт/опционален.
- Профиль: аватар (загрузка/кроп), имя, био, номер телефона (опционально).
- Персонализация: тема (светлая/тёмная/системная), размер шрифта, язык (RU/EN), обои чата (из набора).
- Приватность: кто видит «был(а) в сети», аватар, био (все / контакты / никто).
- Блокировка пользователей (чёрный список).
- Удаление аккаунта.

### 2.2 Базовая коммуникация
- Текстовые сообщения с форматированием: **жирный**, *курсив*, `моноширинный`, ~~зачёркнутый~~, списки, цитаты, спойлер, ссылки (Markdown-подмножество).
- Медиа: фото, видео, аудио, документы (PDF, DOC, ZIP и др.), контакты (vCard), геолокация (точка на карте).
- Голосовые сообщения: запись, волновая форма, ускоренное прослушивание (1x/1.5x/2x). Видео-кружки — фаза 2.
- Реакции эмодзи на сообщения (👍❤️😂😮😢😡 + полный пикер), двойной тап = ❤️.
- Ответ (reply) с цитированием, переход к цитируемому сообщению по клику.
- Пересылка сообщений/медиа в другие чаты, копирование текста.
- Редактирование отправленных (метка «изменено») и удаление «у себя» / «у всех» (у всех — в течение 48 часов).
- Статусы доставки: отправлено (✓), доставлено/прочитано (✓✓) — как на макетах.
- Индикаторы: «печатает…», «записывает голосовое…» (как на макетах), онлайн / «был(а) в сети N минут назад».
- Черновики: сохраняются на сервере, синхронизируются между устройствами.
- Непрочитанные: счётчики на чатах, разделитель «Непрочитанные сообщения» в чате.
- Упоминания @username в группах (подсветка + отдельный счётчик).
- Превью ссылок (Open Graph).
- **Опросы** в группах (анонимные/публичные, мультивыбор).
- Отложенная отправка (запланированные сообщения) — фаза 2.

### 2.3 Группы
- Создание групп: название, аватар, описание.
- Приглашение: прямое добавление из контактов, инвайт-ссылка, QR-код.
- Роли: владелец, администраторы с гранулярными правами (удалять сообщения, банить, добавлять участников, закреплять, менять инфо).
- Закреплённые сообщения (несколько, с каруселью вверху чата).
- Общий медиа-архив чата: вкладки Медиа / Файлы / Ссылки / Голосовые.
- Альбомы (группировка нескольких фото в одно сообщение).
- Лимит участников MVP — 200.
- Публичные группы по ссылке — фаза 2, каналы — фаза 3.

### 2.4 Поиск и организация
- Глобальный поиск: по чатам, контактам, username, сообщениям.
- Поиск внутри чата: по тексту, по дате (календарь), фильтры медиа/ссылки/файлы.
- Закрепление чатов вверху списка, архив чатов, mute (навсегда / на время).
- Сохранённые сообщения (Saved Messages) — личное облако, закладки на сообщения (как на макете — флажок).
- Папки чатов — фаза 2.

### 2.5 Уведомления
- **Web Push** (VAPID): работает на iPhone для PWA, установленной на домашний экран (iOS 16.4+).
- Настройки: звук, превью текста, отключение по чату, режим «не беспокоить».
- Внутренние уведомления (in-app) + бейдж на иконке приложения (Badging API).

### 2.6 Кроссплатформенность и PWA
- Облачная синхронизация: вся история на сервере, доступна с любого устройства, real-time между вкладками/устройствами.
- PWA: `manifest.webmanifest` (иконки всех размеров, `apple-touch-icon`, splash), service worker (кэш оболочки, offline-режим просмотра истории, очередь исходящих при отсутствии сети), `display: standalone`.
- Автономность от телефона: веб-версия самодостаточна.
- Экспорт данных: выгрузка истории чатов в JSON/HTML (для владельца аккаунта; полный экспорт — для администратора сервера).

### 2.7 Безопасность
- Шифрование при передаче: TLS (HTTPS/WSS) везде.
- Пароли — Argon2/bcrypt; коды входа — одноразовые, TTL 10 минут, ограничение попыток.
- Все секреты — только в переменных окружения (`.env` в `.gitignore`, `.env.example` в репозитории). Никаких ключей в коде и в git-истории.
- Rate limiting (вход, отправка сообщений, загрузка файлов), защита от брутфорса кодов.
- Валидация и санитизация всего пользовательского ввода (XSS), CSP-заголовки, CORS-whitelist.
- Загрузка файлов: проверка MIME, лимиты размеров, генерация превью на сервере, раздача с отдельного origin/подписанными URL.
- Блокировка приложения PIN-кодом (локально, на устройстве) — фаза 2.
- **E2EE (сквозное шифрование)** — фаза 3: X25519 + AES-GCM для секретных чатов (по образцу Signal Protocol, упрощённо). В MVP — серверное хранение с шифрованием на уровне диска/БД.

### 2.8 Звонки (фаза 2–3)
- Голосовые вызовы 1-на-1 (WebRTC, STUN/TURN — coturn на VPS).
- Видеозвонки 1-на-1, переключение камер, выключение микрофона/камеры.
- Групповые видеоконференции до 30 участников (SFU — LiveKit/mediasoup) — фаза 3.
- В MVP: UI кнопки звонка присутствует (как на макете), по нажатию — «Звонки скоро появятся».

### 2.9 Стикеры и GIF (фаза 2)
- Поиск GIF через Tenor/GIPHY API.
- Наборы стикеров, создание своих — фаза 3.
- В MVP: полный эмодзи-пикер.

## 3. Нефункциональные требования

- **Производительность:** открытие чата < 300 мс (после загрузки приложения); виртуализация списка сообщений; пагинация истории (по 50 сообщений); lazy-загрузка медиа; Lighthouse PWA ≥ 90.
- **Совместимость:** iOS Safari 16.4+, Chrome/Edge/Firefox последних 2 версий.
- **Масштаб MVP:** до 1 000 пользователей, до 100 одновременных WebSocket-подключений.
- **Доступность:** контраст по WCAG AA, поддержка увеличения шрифта, aria-атрибуты для основных элементов.
- **Локализация:** интерфейс RU (основной) + EN, даты/время по локали.

## 4. Приоритизация (фазы)

| Фаза | Состав | Результат |
|---|---|---|
| 0 | Каркас репозитория, CI, дизайн-система, деплой скелета | Пустое приложение открывается по URL |
| 1 | Auth (email-код, JWT), профиль, username, контакты | Вход и список контактов |
| 2 | Личные чаты: real-time сообщения, статусы, typing, реакции, reply, edit/delete, черновики | Полноценная переписка 1-на-1 |
| 3 | Медиа, голосовые, Saved Messages, поиск, пересылка | Медиа-коммуникация |
| 4 | Группы, роли, закреп, инвайт-ссылки, опросы, медиа-архив | Групповая работа |
| 5 | PWA-полировка: push, offline, iOS-адаптация, тёмная тема, экспорт | **MVP для показа** |
| 6+ | Звонки (WebRTC), E2EE, стикеры, папки, каналы, планируемые сообщения | Пост-MVP |

## 5. Критерии приёмки MVP

1. Пользователь регистрируется по e-mail, задаёт username и аватар.
2. Находит собеседника по @username, ведёт переписку в реальном времени с двух устройств.
3. Отправляет фото, файл, голосовое; ставит реакцию; отвечает цитатой; редактирует и удаляет сообщение.
4. Создаёт группу, приглашает по ссылке, назначает админа, закрепляет сообщение.
5. Устанавливает приложение на домашний экран iPhone, получает push при новом сообщении, интерфейс соответствует макетам (включая тёмную тему).
6. Поиск находит чаты и сообщения; черновики и история синхронизируются между устройствами.
7. Ни одного секрета в репозитории; `docker compose up` поднимает проект локально целиком (БД — PostgreSQL 16, готовый скрипт — `db/schema.sql`).
